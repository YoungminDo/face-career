## 1. 디자인 시스템 비교

### 1.1 컬러 시스템

| 구분 | READ_Career | FACE_Career | 매칭 판정 |
|------|-------------|-------------|----------|
| 창조가/창작형 | R #ffac62 (orange) | Cr #F97316 (orange) | **완벽 매칭** - 동일 계열 오렌지 |
| 상담가/교감형 | E #a2d791 (green) | Em #22C55E (green) | **완벽 매칭** - 동일 계열 그린 |
| 행동가/운영형 | A #fb7980 (red/pink) | Op #3B82F6 (blue) | **색상 불일치** - 빨강 vs 파랑 |
| 분석가/설계형 | D #6eb1ff (blue) | Ar #8B5CF6 (purple) | **색상 불일치** - 파랑 vs 보라 |

**권장안**: READ 컬러를 FACE에 그대로 적용
- 창작형(Cr): `#ffac62` (READ 오렌지 유지)
- 교감형(Em): `#a2d791` (READ 그린 유지)
- 운영형(Op): `#fb7980` (READ 레드/핑크 채택) ← 현재 파랑에서 변경
- 설계형(Ar): `#6eb1ff` (READ 블루 채택) ← 현재 보라에서 변경

각 컬러에 light/dark variant도 같이 적용:
```
창작형: main #ffac62 / light #fff3e8 / dark #e6944a
교감형: main #a2d791 / light #f0f9ed / dark #8bc078
운영형: main #fb7980 / light #fff0f1 / dark #e25f66
설계형: main #6eb1ff / light #eef6ff / dark #5699e6
```

## 3. 일러스트 매칭 검토

### 3.1 READ 캐릭터 일러스트 현황

READ는 각 유형별 4개 서브타입 일러스트를 보유 (캐릭터 + 컬러 카드 형태):

| READ 유형 | 서브타입 | 카드 배경색 | 캐릭터 특징 |
|---------|--------|----------|----------|
| **R 창조가** | RR/RE/RA/RD | 오렌지 (#ffac62) | 갈색 재킷, 오렌지 머리, 스카우트 모자 |
| **E 상담가** | EE/ER/EA/ED | 그린 (#a2d791) | 갈색 재킷, 녹색 머리, 헤드폰/모자 |
| **A 행동가** | AA/AR/AE/AD | 핑크 (#fb7980) | 갈색 재킷, 핑크 포인트 |
| **D 분석가** | DD/DR/DE/DA | 블루 (#6eb1ff) | 갈색 재킷, 파란 머리, 안경 |

**공통 디자인 언어**:
- 동일한 갈색 재킷 베이스
- 유형별 포인트 컬러 (머리색, 모자, 악세서리)
- 전신 캐릭터, 2D 일러스트 스타일
- 카드에 번호(1-4) + 서브타입 코드

### 3.2 FACE 유형과 매칭

| READ → FACE | 개념 매칭 | 컬러 매칭 | 일러스트 적용 |
|-------------|---------|---------|-----------|
| **창조가(R) → 창작형(Cr)** | **완벽** (창의력, 비전, 독립적) | **완벽** (오렌지) | **그대로 사용 가능** |
| **상담가(E) → 교감형(Em)** | **완벽** (공감, 소통, 관계) | **완벽** (그린) | **그대로 사용 가능** |
| **행동가(A) → 운영형(Op)** | **높음** (실행력, 체계, 추진력) | 조정 필요 (핑크→블루 현재) | **컬러 조정 시 사용 가능** |
| **분석가(D) → 설계형(Ar)** | **높음** (분석, 논리, 구조) | 조정 필요 (블루→보라 현재) | **컬러 조정 시 사용 가능** |

### 3.3 서브타입 매칭 (16종)

READ의 16종 서브타입 → FACE의 16종 세부유형:

| READ 서브타입 | READ 이름 | FACE 세부유형 | FACE 이름 |
|-----------|---------|-----------|---------|
| RR 창조가 | 창조가 | CrCr | 순수 창작형 |
| RE 안내자 | 안내자 | CrEm | 교감의 감각을 가진 창작형 |
| RA 승부사 | 승부사 | CrOp | 운영의 체계를 가진 창작형 |
| RD 장인 | 장인 | CrAr | 설계의 정밀함을 가진 창작형 |
| EE 상담가 | 상담가 | EmEm | 순수 교감형 |
| ER 촉진자 | 촉진자 | EmCr | 창작의 영감을 가진 교감형 |
| EA 대변인 | 대변인 | EmOp | 운영의 힘을 가진 교감형 |
| ED 조직자 | 조직자 | EmAr | 설계의 눈을 가진 교감형 |
| AA 행동가 | 행동가 | OpOp | 순수 운영형 |
| AR 전략가 | 전략가 | OpCr | 창작의 감각을 가진 운영형 |
| AE 선도자 | 선도자 | OpEm | 교감의 마음을 가진 운영형 |
| AD 모험가 | 모험가 | OpAr | 설계의 관점을 가진 운영형 |
| DD 분석가 | 분석가 | ArAr | 순수 설계형 |
| DR 세신가 | 세신가 | ArCr | 창작의 열정을 가진 설계형 |
| DE 조정자 | 조정자 | ArEm | 교감의 따뜻함을 가진 설계형 |
| DA 개발자 | 개발자 | ArOp | 운영의 추진력을 가진 설계형 |

**결론**: 16종 서브타입이 1:1로 정확히 대응되므로 **일러스트 직접 재사용 가능**

## 4. 적용 가능한 READ 디자인 요소

### 4.1 섹션 커버 패턴 (다크 + 컬러 원)
```
배경: #2c2c2c (다크 그레이)
중앙: 445px 컬러 원 (유형 메인 컬러)
텍스트: "Part N. 서브타이틀" + "메인 타이틀" (48px/900w)
하단: DASH 로고 + 태그라인
```
→ FACE 각 섹션(Focus, Anchor, Capacity, Energy) 도입 페이지에 적용

### 4.2 설명 페이지 패턴 (다크 BG + 큰 타이틀)
```
배경: #2c2c2c
타이틀: 54px/900w 화이트
라벨: "설명" (유형 컬러)
본문: 12px/300w 라이트그레이(#f3f4f6), 양쪽정렬
```
→ FACE "나다움이란", "역량이란" 등 설명 페이지에 적용

### 4.3 콘텐츠 페이지 패턴
```
배경: #ffffff 또는 #f3f4f6
헤더: 18px/800w (유형 컬러 라벨 우측)
컨텐츠: 카드 컴포넌트, 2열 그리드
테이블: 유지/강화/보완 3열
푸터: "READ" + 태그라인 (absolute bottom 50px)
```
→ FACE 역량 분석, 직무 적합도 페이지에 적용

### 4.4 차트 컴포넌트
| READ 차트 | FACE 적용 | 난이도 |
|----------|---------|------|
| 라인 그래프 (4유형 점수) | Focus 4유형 점수 표시 | **쉬움** - 동일 구조 |
| 레이더/헥사곤 차트 (역량 비교) | Capacity 역량 비교 | **중간** - SVG 생성 |
| 바 차트 (역량 개별) | 이미 존재 | **쉬움** |
| 게이지 차트 (지수 표시) | Energy 동기/행동 지수 | **쉬움** |
| 회복탄력성 그래프 | 불필요 | - |

### 4.5 캐릭터 카드 패턴
```
좌측: 유형 아이콘 + 유형명 + 서브타입 목록
우측 상단: 메인 서브타입 일러스트 (큰 카드)
우측 하단: 나머지 3개 서브타입 (작은 카드 2개)
카드 배경: 유형 메인 컬러
번호: 좌상단 원형 번호
```
→ FACE "내 유형" 소개 페이지에 적용

## 6. 일러스트 적용 전략

### 6.1 직접 사용 가능한 일러스트

| READ 일러스트 | 위치 | FACE 적용 |
|------------|------|---------|

| logo-group47.svg | 모든 페이지 푸터 | **그대로 사용** |

### 6.2 라벨 변경이 필요한 부분

| 현재 (READ) | 변경 (FACE) |
|-----------|-----------|
| 창조가 REVOLUTIONER | 창작형 CREATOR |
| 상담가 EMOTIONER | 교감형 CONNECTOR |
| 행동가 ACTIONER | 운영형 SUPPORTER |
| 분석가 DETECTOR | 설계형 ARCHITECT |
| RR, RE, RA, RD | CrCr, CrEm, CrOp, CrAr |

### 6.3 캐릭터 일러스트 재사용 조건

1. **이미지 자체**: 색상 매칭이 이미 맞으므로 PNG 그대로 사용 가능
2. **라벨/텍스트**: HTML/CSS 레이어에서 변경 (이미지 위의 텍스트)
3. **카드 배경색**: CSS에서 유형 컬러 변수로 제어 → 자동 적용
4. **서브타입 코드**: HTML 텍스트만 변경 (RR→CrCr 등)

**결론: 일러스트 이미지 파일 수정 없이 텍스트/CSS만 변경하여 적용 가능**

**Option B: 점진적 마이그레이션**

## 8. 최종 판정

### 적용 가능성: **매우 높음 (95%)**

| 항목 | 판정 | 비고 |
|------|------|------|
| 컬러 시스템 | **적용 가능** | 2종 완벽매칭, 2종 READ 컬러 채택으로 해결 |
| 타이포그래피 | **적용 가능** | 동일 폰트(Pretendard), 액센트 폰트 추가만 |
| 레이아웃/페이지 구조 | **적용 가능** | 동일 A4, 유사 여백 |
| 섹션 커버 (다크+원) | **적용 가능** | HTML/CSS만으로 구현 |
| 캐릭터 일러스트 | **적용 가능** | 유형/컬러 1:1 매칭, 텍스트만 변경 |
| 차트 컴포넌트 | **적용 가능** | 라인/레이더/바/게이지 모두 구현 가능 |
| 콘텐츠 구조 | **적용 가능** | READ 10 Part → FACE 5 Part로 압축 |

### 주의사항
1. **운영형/설계형 컬러 변경** 시 기존 FACE UI(웹)도 함께 변경 필요
2. **일러스트 라이선스**: READ와 FACE가 같은 조직(DA-SH) 소속이므로 문제 없음
3. **서브타입 이름**: FACE 고유 네이밍 적용 (READ 이름 그대로 쓰지 않음)
4. **Part 7~10 (회복탄력성/커뮤니케이션/이미지/생산성)**: FACE에 해당 데이터 없으므로 생략
