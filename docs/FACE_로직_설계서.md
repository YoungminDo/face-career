# FACE 커리어 진단 — 로직 설계서

> 모든 산출 로직의 what, why, how를 기록.
> 변경 시 이 문서를 먼저 업데이트하고 코드에 반영할 것.

## 1. Focus 점수 산출

### What
18문항 × 4선택지(Co/Cr/Su/Ar) 랭킹 → 4유형 점수 산출

### How
```
각 문항: 1위=4점, 2위=3점, 3위=2점, 4위=1점
Co 점수 = 18문항에서 Co가 받은 점수 합 (범위: 18~72)
Cr/Su/Ar도 동일
4개 합 = 항상 180 (강제분배, ipsative)
```

### Why
- **랭킹(ipsative)을 쓰는 이유**: 리커트로 하면 "다 좋아요" 응답 가능 (동의편향). 랭킹은 반드시 차이를 만들어야 하므로 진짜 선호가 드러남
- **합이 항상 180인 이유**: 강제분배 특성. "Connector가 높으면 다른 게 낮아짐"은 제로섬이라는 뜻이 아니라, 상대적 선호 순서를 측정한다는 뜻
- **18문항인 이유**: 6개 맥락 × 3문항 = 한 유형당 18번 측정. 심리측정에서 construct당 최소 5문항 권장 → 18번이면 충분

### 주기능/부기능 결정
```
4개 점수 내림차순 정렬
주기능 = 1위
부기능 = 2위
세부유형코드 = 주기능 + 부기능 (예: "CoCr")
```

### Why
- Holland RIASEC도 동일 방식 (1위 코드 + 2위 코드)
- 주기능 하나만 쓰면 4유형뿐이지만, 부기능까지 합치면 16유형 → 해상도 4배

## 2. Focus 정제 (동점 처리)

### What
상위 2개 유형 점수 차이 ≤ 3점일 때 추가 7문항으로 주기능 확정

### How
```
활성화 조건: |1위 점수 - 2위 점수| ≤ 3
7문항 각각 단일선택 → 선택된 유형에 +1점
상위 2개 중 정제 점수가 높은 쪽이 주기능
```

### Why
- **≤3점 기준**: 18문항 랭킹에서 1점 차이는 문항 1개의 순위가 바뀌면 역전됨. 3점까지는 측정 오차 범위 내. 정규분포 가정 시 약 25~30%의 응답자가 해당
- **7문항인 이유**: 상황 기반 3문항 + 선호 기반 3문항 + 최종 결정 1문항. 홀수여야 동점 발생 안 함
- **단일선택인 이유**: 랭킹 대신 단일선택을 쓰면 "가장 나다운 것"에만 집중하므로 결정력이 높음

## 3. Anchor 점수 산출

### What
Part 1 리커트 6문항 + Part 2 강제선택 6문항 + Part 3 관심영역 6문항 → 가치관 프로파일 + 관심영역

### How — Part 1 (가치관 리커트)
```
6개 앵커 각각 1문항 (1~7점)
mastery, growth, autonomy, stability, purpose, balance
```

### How — Part 2 (가치관 강제선택)
```
6쌍의 trade-off. 각 앵커가 정확히 2번 등장.
승자 = +2점, 패자 = +0점
각 앵커 범위: 리커트(1~7) + 강제(0~4) = 1~11
```

### Why
- **리커트 + 강제선택 혼합인 이유**: 리커트만 쓰면 "다 중요해요" 응답 가능. 강제선택은 "하나를 포기해야 할 때" 진짜 우선순위가 드러남. 둘을 합쳐야 완전한 가치관 프로파일
- **각 앵커 2번 등장하는 이유**: 6개 앵커에서 2개씩 짝짓기 = C(6,2)=15쌍 가능. 전부 하면 15문항이라 길어짐. 각 앵커 2번이면 균등하면서도 6문항으로 충분
- **리커트가 60%, 강제가 40% 비중인 이유**: 리커트(1~7, 범위 6)가 강제선택(0~4, 범위 4)보다 비중이 큼. 기본 선호도는 리커트로 잡고, 미세 차이는 강제선택으로 보정

### How — Part 3 (관심영역)
```
6문항 × 4선택지 랭킹 = 24슬롯
8개 관심방향 × 3회 균등출현 = 24 (완벽 균등분배)
점수: 1위=4점, 4위=1점 (Focus와 동일 방식)
```

### Why
- **별도 체크박스가 아니라 진단 문항인 이유**: "IT에 관심있어요" 같은 자기보고는 유행에 휩쓸림. "어떤 문제를 풀고 싶은가?"로 묻으면 관심의 본질이 드러남. 산업은 바뀌지만 관심의 방향은 비교적 안정적
- **8개 방향인 이유**: 산업을 직접 나열하면 20개+ 되어 복잡. "기술/사회/비즈니스/크리에이티브/헬스/환경/교육/조직" 8방향이 대부분의 산업을 커버하면서도 직관적

## 4. 나다움 역량 매핑 (Focus + Anchor 이중 구조)

### What
32개 역량 각각에 대해 🟢강한 나다움/🟡나다움/🔴후천적을 판정

### How
```
1) Focus 차원별 역량 (일하는 방식에서 오는 나다움)
   People:      공감, 협상, 코칭, 네트워킹, 고객통찰, 조직통찰 (6개)
   Task:        분석, 수리논리, 전략적사고, 기술이해, 시장통찰 (5개)
   Exploring:   창의, 스토리텔링, 경험설계, 관점전환, 변화적응 (5개)
   Structuring: 정확성, 책임감, 프로세스관리, 프로세스설계, 평가력 (5개)

2) Anchor 가치관별 역량 (가치관에서 에너지를 쏟는 나다움)
   전문성: 분석, 수리논리, 기술이해, 전략적사고 (4개)
   성장:   목표지향, 의사결정, 효율성추구 (3개)
   자율:   창의, 관점전환, 변화적응, 프로세스설계 (4개)
   안정:   정확성, 책임감, 프로세스관리, 비용관리, 자금운용, 계약관리, 제도설계 (7개)
   기여:   공감, 코칭, 고객통찰, 인재발굴 (4개)
   균형:   일정관리, 효율성추구, 비용관리 (3개)

3) 판정
   Focus에 있고 + Anchor에도 있음 → 🟢 강한 나다움
   Focus에만 있음                 → 🟡 나다움(성향)
   Anchor에만 있음                → 🟡 나다움(가치)
   둘 다 없음                     → 🔴 후천적
```

### Why
- **Focus 차원별 역량 수가 균등하지 않은 이유(5~6개)**: 8개씩 억지로 맞추면 자연스럽지 않은 매핑이 들어감. "비용관리가 Task 나다움"이라고 하면 사용자가 납득 못함. 자연스러운 것만 넣었더니 5~6개씩
- **나머지 9개를 Anchor가 잡아주는 이유**: Focus로 안 잡히는 역량(의사결정력, 목표지향성, 일정관리 등)은 성격이 아니라 가치관에서 에너지를 쏟아서 발달하는 역량. "성장을 중시하면 자연스럽게 목표를 세우게 됨"
- **이중 나다움(🟢)이 있는 이유**: Focus와 Anchor가 같은 역량을 가리키면 성향+가치관이 모두 뒷받침 → 가장 자연스럽고 에너지 소모 최소. 리포트에서 "당신의 가장 강력한 무기"로 표현
- **30개 전부 커버(MECE)인 이유**: Focus 21개 + Anchor 24개(중복 포함) = 합치면 30개 전부. 어떤 역량도 "영구 후천적"이 아님. Focus×Anchor 조합에 따라 나다움이 달라짐

## 5. Capacity 점수 산출

### What
24문항 랭킹 → 32개 역량 원점수 → 0~100 환산점수

### How
```
24문항 × 4선택지 = 96슬롯
32역량 × 3회 출현 = 96 (완벽 균등분배)
각 선택지: 특정 역량 코드에 매핑
점수: 1위=4점, 2위=3점, 3위=2점, 4위=1점

역량별 원점수: 3~12 (3문항 × 1~4점)
환산: (원점수 - 3) / 9 × 100 = 0~100점
```

### Why
- **0~100 환산인 이유**: 원점수 3~12는 직관적이지 않음. 0~100이면 "70점이면 괜찮다"는 직관이 통함
- **32역량 × 3회의 신뢰도 한계**: 3문항으로 하나의 역량을 측정하면 Cronbach alpha ≈ 0.4~0.5. 개별 역량 점수는 "경향성 참고"로만 사용. 직무 적합도는 가중평균으로 계산하여 오차를 희석
- **4개 클러스터(Insight/Drive/Connect/Create) 평균**: 개별 역량보다 클러스터 평균이 안정적. 클러스터당 5~11개 역량 → 실질적으로 15~33문항 측정과 유사한 안정성

## 6. 직무 적합도 산출 (연속 점수)

### What
47개 직무별 8개 핵심역량 × 나의 점수 → 연속적 적합도 % (0~100)

### How
```
Step 1: 나다움 보정
  각 역량의 나다움 레벨에 따라 최소 점수 보장:
    🟢 강한 나다움 → adjustedScore = max(실제점수, 65)
    🟡 나다움      → adjustedScore = max(실제점수, 50)
    🔴 후천적      → adjustedScore = 실제점수 (보정 없음)

Step 2: 가중치 적용
  핵심역량 순서에 따른 가중치:
    역량 1~2: × 1.3 (가장 핵심)
    역량 3~4: × 1.1
    역량 5~6: × 1.0
    역량 7~8: × 0.8 (보조적)

Step 3: 가중평균
  적합도 = Σ(adjustedScore × weight) / Σ(weight)
```

### Why — 계단식(Pass/Fail)에서 연속식으로 바꾼 이유
- **기존 문제**: 60점 기준 Pass/Fail → 59점과 60점 사이에 13% 점프. 비현실적
- **해결**: 실제 점수를 반영하면 72%, 84%, 63% 같은 자연스러운 연속 수치가 나옴. 사용자가 결과를 더 신뢰함

### Why — 나다움 보정을 하는 이유
- **문제**: 나다움 역량인데 아직 경험이 없어서 점수가 낮은 경우, 적합도가 과소평가됨
- **예시**: Connector인데 코칭 역량이 35점 → 코칭은 People 차원 나다움이라 잠재력이 있지만, 대학생이라 경험이 없어서 낮게 나온 것
- **보정 효과**: max(35, 50) = 50으로 올림 → "잠재력이 있으니 완전 0은 아님"
- **강한 나다움(65)과 나다움(50)의 차이**: 성향+가치관 둘 다 뒷받침하면 잠재력이 더 높으므로 floor를 65로
- **후천적은 보정 없음**: 성향에도 가치관에도 안 맞으면 실제 점수 그대로. 정직하게 반영

### Why — 가중치를 쓰는 이유
- **문제**: 직무별 8개 역량이 모두 동일하게 중요하지 않음. 영업의 "커뮤니케이션"과 "비용관리"는 중요도가 다름
- **해결**: 핵심역량 순서(1번이 가장 중요)에 가중치 부여. 매핑 테이블에서 역량 순서가 이미 중요도순으로 배열되어 있으므로 추가 데이터 불필요
- **1.3~0.8 범위인 이유**: 차이가 너무 크면 8번 역량이 의미 없어지고, 너무 작으면 가중치 효과가 없음. 1.3/0.8 = 1.625배 차이가 적절

## 7. 나다움 적합도 (Focus×Anchor → 직무 자연스러움)

### What
Capacity 진단 없이, Focus 유형과 Anchor 가치관만으로 "이 직무가 나에게 자연스러운가"를 판정

### How
```
직무의 8개 핵심역량 중 나다움(🟢 또는 🟡) 역량이 몇 개인지 계산
나다움 적합도 = 나다움 역량 수 / (8 - Foundation 수) × 100

예: BD의 8개 역량 중 Foundation 2개 제외 = 6개 판정 대상
    그 중 나다움 4개 → 나다움 적합도 = 4/6 = 67%
```

### Why
- **Capacity 없이도 의미 있는 이유**: 나다움 적합도는 "잘하는가"가 아니라 "자연스러운가"를 봄. 능력 측정 전에 방향성을 알 수 있음
- **활용**: 리포트 직무 추천에서 2×2 매트릭스의 한 축으로 사용

## 8. 직무 추천 2×2 매트릭스

### What
나다움 적합도(자연스러운가) × 역량 적합도(잘하는가) = 4가지 상태

### How
```
                    나다움 적합도 ≥ 60%       나다움 적합도 < 60%
                ┌────────────────────┬────────────────────┐
역량 적합도      │  🟢 Sweet Spot     │  🟡 후천적 강점     │
≥ 60%           │  자연스럽고 잘함     │  잘하지만 에너지↑   │
                │  → 최우선 추천       │  → 소진 위험 경고   │
                ├────────────────────┼────────────────────┤
역량 적합도      │  🔵 잠재력          │  🔴 비추천          │
< 60%           │  나답지만 아직 미숙   │  안 맞고 못함       │
                │  → 개발하면 빠르게↑  │  → 피하는 게 낫다   │
                └────────────────────┴────────────────────┘
```

### Why
- **기존 문제**: 역량 적합도 하나만 보면 "잘하지만 소진되는 직무"와 "잘하고 자연스러운 직무"를 구분 못함
- **🟡 후천적 강점이 핵심**: "잘하는데 왜 힘들지?"라는 직장인의 고민에 대한 답. 역량 점수는 높은데 나다움이 아니면 에너지가 많이 든다는 경고
- **🔵 잠재력이 Expandable의 근거**: 나다움인데 점수가 낮은 건 "아직 안 키운 것"이지 "못하는 것"이 아님. 개발하면 빠르게 올라감 → Expandable 직무로 추천

## 9. Expandable 직무 판정

### What
Core Fit에는 안 들지만, 역량 개발하면 확장 가능한 직무 추천

### How
```
조건:
  1) Core TOP 5에 포함되지 않은 직무
  2) 현재 역량 적합도 ≥ 40% (너무 낮으면 제외)
  3) 부족 역량의 총 갭 ≤ 40점 (현실적으로 메울 수 있는 범위)

정렬: 총 갭이 작은 순 (확장이 쉬운 것부터)

추가 정보:
  - 부족 역량별 갭 크기
  - 각 역량의 나다움 레벨 (나다움이면 개발 속도 빠름)
  - 예상 노력: 갭 ≤10 "단기", ≤25 "중기", else "장기"
```

### Why
- **총 갭 ≤40인 이유**: 역량 1개 갭이 40이면 거의 처음부터 시작. 2개 역량이 각 20씩이면 현실적 개발 범위. 40 초과면 "Expandable"이 아니라 "먼 미래"
- **나다움 역량 갭부터 메우는 이유**: 나다움 역량은 투자 대비 성장이 빠름. 20점 갭이라도 나다움이면 3개월, 후천적이면 1년 걸릴 수 있음

## 10. 직무 벤치마크 (S급/A급/평균)

### What
특정 직무의 핵심역량별로 S급/A급/평균 기준과 비교

### How
```
Phase 1 (이론적 벤치마크):
  S급: 85점 (상위 10% 추정)
  A급: 70점 (상위 30% 추정)
  평균: 55점

Phase 2+ (실데이터):
  해당 직무 종사자 진단 데이터 축적 → 실제 백분위 기반 재설정

종합 등급:
  S급 역량 4개+ → S급
  A급 이상 역량 4개+ → A급
  A급 이상 역량 2~3개 → 평균 이상
  그 외 → 개발 필요

개발 우선순위:
  나다움 역량의 갭부터 메움 (빨리 올라가니까)
  → 후천적 역량의 갭은 나중에
```

### Why
- **이론적 벤치마크로 시작하는 이유**: 실데이터가 없어도 전문가 추정치로 의미 있는 비교 가능. "이 직무에서 상위 30%가 되려면 이 역량이 70점 이상이어야 합니다"는 방향성을 줌
- **S/A/평균 3단계인 이유**: 5단계(S/A/B/C/D)는 학교 성적 같아서 부정적. 3단계는 "어디쯤인지"만 알려주고, 구체적 개발 방향을 제시
- **나다움 순서로 개발하는 이유**: 같은 20점 갭이라도 나다움 역량은 성향/가치관이 뒷받침하므로 학습 동기가 높고 습득이 빠름. ROI가 높은 것부터 개발

## 11. Anchor → 기업 카테고리 매칭

### What
6개 앵커 점수 → 9개 기업 카테고리와 코사인 유사도 계산

### How
```
사용자 앵커 벡터 = [mastery, growth, autonomy, stability, purpose, balance] 정규화(0~100)

9개 카테고리 고정 벡터:
  공무원:     [30, 20, 15, 95, 60, 85]
  공공기관:   [40, 30, 20, 90, 70, 80]
  대기업:     [50, 70, 30, 80, 40, 50]
  외국계:     [70, 75, 65, 60, 40, 70]
  중견기업:   [55, 55, 45, 65, 45, 60]
  강소기업:   [85, 50, 50, 55, 40, 50]
  스타트업:   [55, 90, 85, 15, 50, 20]
  사회적기업: [40, 45, 55, 30, 95, 55]
  프리랜서:   [70, 60, 95, 10, 45, 75]

코사인 유사도 = dot(user, category) / (|user| × |category|)
```

### Why
- **코사인 유사도인 이유**: 유클리드 거리는 "전체 크기"에 민감. 앵커 점수가 전반적으로 높은 사람과 낮은 사람이 다르게 나옴. 코사인은 "방향(비율)"만 보므로, "어떤 앵커를 상대적으로 중시하는가"에 집중
- **카테고리 벡터가 고정인 이유**: Phase 1에서는 전문가 추정. Phase 2+에서 잡플래닛/블라인드 데이터로 보정 가능
- **조직유형과 문화를 하나의 벡터로 보는 이유**: 앵커 프로파일이 곧 조직 문화. "안정 90 + 균형 85 = 공무원"은 조직유형이면서 동시에 "안정적이고 워라밸 좋은 문화"를 의미. 별도 문화 차원 불필요

## 12. Energy 점수 산출

### What
12문항 리커트 → 동기충족도/몰입도/행동전환도 → 성장단계 9칸 → 준비도 3단계

### How
```
동기충족도 = (E-01~E-05 합 / 35) × 100 (%)
  E-07은 역채점: 8 - 원점수
몰입도 = (E-06 점수 / 7) × 100
행동전환도 = (E-08~E-12 합 / 35) × 100

성장단계 매트릭스:
  동기: ≥70% 충족, 40~69% 부분충족, <40% 미충족
  행동: ≥50% 높음, 30~49% 보통, <30% 낮음
  → 3×3 = 9칸

준비도:
  🟢 도전가능: 성장활성기 / 균형조율기 / 전환가속기
  🟡 탐색단계: 잠재축적기 / 탐색진행기 / 돌파시도기
  🔴 회복우선: 방향설정기 / 에너지충전기 / 기반구축기
```

### Why
- **E모듈이 맨 앞(워밍업)에 오는 이유**: 리커트라서 빠르고 쉬움. F/A/C 후에 하면 피로 효과로 에너지 점수가 낮게 왜곡됨. 스크리닝 목적
- **준비도가 추천의 "게이트"인 이유**: 🔴인 사람에게 "지금 지원하세요!"는 무책임. Energy가 추천의 차원이 아니라 "지금 할 수 있는가"의 필터 역할
- **9단계를 3단계로 축약하는 이유**: 9칸 매트릭스는 전문가용. 사용자에게는 🟢🟡🔴 신호등으로 충분. 상세는 리포트에서

## 13. 전체 추천 파이프라인

### 최종 흐름
```
1) Focus(유형) × Anchor(가치관+관심) → 나다움 역량 맵 결정
2) Capacity(역량 점수) 측정
3) 각 직무별:
   a) 나다움 적합도 = 8개 핵심역량 중 나다움 비율
   b) 역량 적합도 = 8개 핵심역량 가중평균 (나다움 보정 포함)
   c) 2×2 매트릭스 배치 (Sweet Spot / 잠재력 / 후천적강점 / 비추천)
4) Anchor → 기업 카테고리 매칭 (코사인 유사도)
5) 관심영역 → 산업 매칭
6) Energy → 준비도 게이트
7) 결과:
   Core: 나다움↑ × 역량↑ 직무 + 매칭 기업카테고리 + 매칭 산업
   Expandable: 나다움↑ × 역량↓ 직무 + 갭 분석
   경고: 나다움↓ × 역량↑ 직무 = 소진 위험
```

### Why 이 순서인 이유
- Focus와 Anchor가 **방향**을 정함 (나다움 = 어디로 갈 것인가)
- Capacity가 **현재 위치**를 측정함 (지금 어디에 있는가)
- 나다움 × 역량 교차가 **전략**을 만듦 (어떻게 갈 것인가)
- Energy가 **타이밍**을 결정함 (지금 갈 것인가)
