# 나다움 구조 변경 (2026-03-18)

## 변경 전
- Focus 4차원(People/Task/Exploring/Structuring)에 역량 배정 → 주기능 2차원 합산 = 11개 나다움 (37%)
- Anchor 6가치관에도 역량 배정 → 합치면 15개 나다움 (50%)
- 대각선 조합(CaSt, ExGu)은 21개 나다움 (70%) → 변별력 없음
- 나다움 보정: max(점수, 50~65) → 적합도가 인위적으로 올라감

## 변경 후
- Focus **유형에 직접** 30개 역량 배정 (차원이 아니라 유형별)
- 촉매자 8개, 탐험가 7개, 수호자 8개, 전략가 7개 = 30개 전부 MECE
- **주기능만** 나다움 판정 (부기능 미사용) → 7~8개 나다움 (23~27%)
- Anchor는 역량 매핑 **완전 제거** → 조직/문화 매칭 전용
- 적합도 보정 **완전 제거** → 순수 점수만 반영

## 이유
1. 주기능+부기능 합산 시 대각선 조합(CaSt)은 4차원 전부 커버 → 변별력 상실
2. Anchor에 역량 매핑하면 "가치관이 높으면 역량도 높다"는 잘못된 전제
3. 나다움 보정(max 50)이 적합도를 인위적으로 올려 비추천 직무도 "보통"으로 바뀜
4. 나다움이 50%이면 대부분의 직무에서 절반이 나다움 → "다 잘 맞아요" 문제

## 새 구조에서의 역할
- Focus: "어떤 역량이 편한가" → 나다움 7~8개 → 직무 방향 필터
- Anchor: "어떤 조직이 맞는가" → 코사인유사도 → 조직/문화 매칭 (역량 무관)
- Capacity: "실제로 얼마나 잘하는가" → 순수 점수 → 적합도 %
- Energy: "지금 도전할 수 있는가" → 준비도 게이트

## 검증
- 나다움 25% → 직무별 나다움 비율 0~63% 분포 → 변별력 강함
- 적합도 순수 점수 → BD 68% vs 회계 35% → 정직한 차이
- 깔때기: 47직무 → F(13개) → A(3조직) → C(5직무) → 최종 3개
